Python学习笔记2：Python基础

发表于 2017-09-30 | 分类于 Programming

这一部分是Python方面的一些学习笔记，用于记录自己学习Python的过程，同时也是对自己Python这块知识的一个梳理，后期也可用于Python复习。

Python学习的第二篇，介绍Python基础知识。

数据类型

Python中的数据类型包括以下几种：

  • 整数 int
  • 浮点数 float
  • 字符串 str
  • 布尔值 （只有TRUE/FALSE两种值）
  • 空值 None
  • 变量 （英文、数字和_的组合）

需要说明的是，变量不能以数字为开头。

除了上述数据类型外，整数部分还包括一种长整数，当整数大于2,147,483,647或小于-2,147,483,647时，python输出会带L标记为长整数（通过print打印出来便于用户理解不会显示）。

同时，在Python中也可显示十六进制和八进制。在输入时，十六进制和八进制首位数字都是0。

字符转义

转义字符为\，当涉及到'（引号）、\本身或者一些\n换行符\t制表符等符号时，需要用到转义字符。转义字符能够保证跟在转义字符\后的字符正常输出，而不被python误判。

常见转义字符：

  • \b 退格（Backspace），等于删除在\b前一个字符
  • \000 空
  • \n 换行
  • \t 横向制表符
  • \v 纵向制表符
  • \r 回车 猪：当两个字符串中间以\r连接时，功效等于后一个字符串替换前一个字符串相应位置
  • \f 换页

当语句中很多字符都涉及到转义时，为了避免添加太多\转义符，可以在字符串之前添加r表示该语句所有字符串默认不转义。

当字符串有很多行时，也可以采用'''...'''格式来优化。

字符串

Unicode字符串

常量中，有一种类型为Unicode字符串（一种标准字符串编码格式），Python普通字符在内部是以8位的ASCII码存储的，而Unicode字符串则存储为16位Unicode字符，这样能够表示更多的字符集。在Python3.0中，所有字符串都是Unicode字符串。

字符串的两种机制

这实际上涉及到值被转化为字符串的两种机制。可以通过str和repr来阐述这两种机制。str函数会把值转化为合理性使得字符串，以便用户理解，repr函数会创建一个字符串，以合法的python表达式的形式表示值。

repr(x)也可以写作`x`实现（反引号，非单引号）。如果希望打印一个包含数字的句子，这时候就起到作用了。

需要说明的是，str和int、float一样，是一种类型，而repr则是一种函数。

字符串编码

对于字符串的处理，存在不同的编码方式，Unicode是一种统一编码方式。常见的编码方式包括Unicode编码、ASCII编码、UTF-8编码、GB2312编码（中文编码）。ASCII编码占一个字节，Unicode编码则占2个字节，可以看到，在存储空间上，Unicode编码比ASCII编码要多一倍的存储空间，因此在Unicode的基础上衍生了“可变长编码”的UTF-8。

UTF-8编码把一个Unicode字符根据不同的数字大小编码成1-6个字节，常用的英文字母被编码成1个字节，汉字通常是3个字节，只有很生僻的字符才会被编码成4-6个字节。当传输的文本包含大量英文字符时，用UTF-8编码能节省空间。

计算机系统通用的字符编码工作方式如下：

  • 在计算机内存中，统一使用Unicode编码，当需要保存到硬盘或者需要传输的时候，就转换为UTF-8编码。
  • 用记事本编辑的时候，从文件读取的UTF-8字符被转换为Unicode字符到内存里，编辑完成后，保存的时候再把Unicode转换为UTF-8保存到文件。
  • 浏览网页的时候，服务器会把动态生成的Unicode内容转换为UTF-8再传输到浏览器。

Python2与Python3在字符串编码上有一定不同，主要是在中文处理上，如ord('中')在Python3上处理输出为20013,而在Python2上则会因为字符问题而报错，ord函数和’chr’函数。

格式化

格式化是指输出格式化的字符串。如’亲爱的xxx你好！你xx月的话费是xx，余额是xx’，这类字符串中的“xxx”内容是根据变量变化的，可以通过格式化来输出结果。实现方式是通过%格式化字符串来实现。

常见的占位符（格式化字符串）有：

  • %d 整数
  • %f 浮点数
  • %s 字符串
  • %x 十六进制数

当只有单个需要替换时，可以不用括号，涉及到多个需要添加括号，并且按照顺序写入。

当不确定应该用以上哪种占位符时，可以一律采用%s，它会把任何数据类型转化为字符串。当涉及到普通字符“%”时，采用转义即可。

list列表

list是一种有序的集合，可以随时添加和删除其中的元素。需要注意的是，list列表时可变的，这点会在之后重点说明。

元组tuple

元组tuple和列表list一样，也是一种序列。唯一不同的是元组tuple不能修改。

元祖tuple的创建

多个元素的元组，可以直接以逗号分隔即可，也可在所有元素外面加上()，单个元素的元组，需要添加一个, 表示这是一个元组。有,的表示元组。

tuple操作

元组tuple操作并不多，基本按照list基本操作（不包括list扩展方法如append等）和基本序列操作。包括：对序列操作并将其转化为元组、基本序列操作。

需要注意的是，当tuple里包含list列表时，当list列表内容变化时，看起来tuple也变化了，但实际tuple并没有变化，tuple指向的序列并不变，变的只是指向的list列表的元素。

元组tuple的意义

有以下两个重要的原因，使得元组tuple是不可替代的。

  • 元组tuple可以在映射（和集合的成员）中当做键使用——而列表list则不行
  • 元组tuple作为很多内建函数和方法的返回值存在，也就是说你必须对元组进行处理。只要不尝试修盖元组，那么处理“元组”绝大多数情况下就是把它们当做列表来进行操作（除非需要使用一些元组没有的方法，例如index和count)

一般来说，列表可能更能满足对序列的所有需求。

文件

注意每次打开文件之后，需要使用f.close()关闭文件，释放存储空间。所以上述语句正确表达应为：

为了更加简化处理，可以直接使用下述表述方式，这样就不用在最后声明f.close()了。

打开文件涉及到以下的操作模式：

  • r 只读模式（默认模式，文件必须存在，不存在则抛出异常）
  • w 只写模式（不可读，不存在则创建，存在则清空内容）
  • x 只写模式（不可读，不存在则创建，存在则报错） python2中没有这个用法
  • a 追加模式（可读，不存在则创建，存在则只追加内容）
  • r+/w+/x+/a+ 表示可同时读写某个文件
  • b表示以字节的方式操作 rb/wb/xb/ab，/r+b/w+b/x+b/a+b

注：以b方式打开时，读取到的内容是字节类型，写入时也需要提供字节类型，不能指定编码

file文件对象方法

  • file.read([size]) 方法用于从文件中直接读取字节，最多读取 size 个字节数据，如果不指定则读取全部。
  • file.readline([size]) 方法则用于从文件中读取一行，每行的长度大于 size 时只读取 size 长度的不完整行。size表示选取行的长度
  • file.readlines([size]) 读取文件的行并返回一个字符串列表，别忘了最重要的方式，用 for 循环直接迭代文件，文件对象是它自己的迭代器。
  • file.write(‘str’) 用于将字符串写入到文件中
  • file.writelines(‘seq’) 用于写入多行，要注意的是换行符是不会自动添加的。
  • file.seek(offset[, whence]) 跟 C 语言中的 fseek() 很相似，offset 可以是正负数表示偏移量，whence 表示相对的位置，值必须是 os.SEEK_SET os.SEEK_CUR os.SEEK_END 中的一个。
  • file.tell() 方法获得当前所在的位置。位置是以字节进行测量的。
  • file.close() 可以关闭文件，或者当文件的引用为 0 时自动关闭。
  • file.fileno() 获得文件对象的文件描述符，可供 os.read() 使用。
  • file.flush() 可以将缓冲中的数据刷新到磁盘上。
  • file.isatty() 用于判别文件是否是 tty 设备。
  • file.truncate([size]) 将文件截取到当前位置，如果提供 size 参数，则截取到 size 大小。

文件属性

  • file.closed 文件是否关闭
  • file.encoding 文件的编码，如果没有则为系统默认编码
  • file.mode 访问文件的模式字符串
  • file.name 文件名字
  • file.newlines 当 没有开始读取文件时为 None ，随着读取把不同的换行符添加到元组中
