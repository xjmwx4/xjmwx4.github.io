以下是个人的回答
1.为什么要做数据分析？
有以下几个原因：
原因1：未来是一个信息与数据时代，大数据在科技领域、社会企业和社会生活方面扮演着尤为重要的角色；
原因2：目前从事的即为数据分析（初级），进一步从事技术型的数据分析，能够得到较好的职业发展，也能得到较为快速的行业入门；

2.对于大数据行业的职业规划？

在知乎有人对数据分析涉及的三个面做了一个说明，即分为BI (Business Intelligence), QA (Quantitative Analyst / Data Scientist), 以及BA (Business Analyst)三类。（详见如何快速成为数据分析师？——王曦的回答）

以上三个方向具体划描述与侧重为：
Business Analyst 定义问题，分析数据，提出并部分实现解决方案；
Quantitative Analyst 定义模型和架构，实现完整的解决方案；
Business Intelligence 对数据和度量进行可视化展示、自动化计算和更新。

SQL是目前数据分析行业的一种通用性基础性语言，结构化查询的使用，使得快速处理大量/海量数据更加便捷…….

SQL正文

SQL简介

SQL是结构化查询语言（Structured Query Language）的缩写。SQL是一种专门用来与数据库沟通的语言，设计SQL的目的是很好地完成一项任务——提供一种从数据库中读写数据的简单有效的方法。

SQL有如下的优点：

  • SQL不是某个特定数据库供应商专有的语言。几乎所有重要的DBMS都支持SQL，所以学习此语言使你几乎能与所有数据库打交道。
  • SQL简单易学。它的语句全都是由有很强描述性的英语单词组成，而且这些单词的数目不多。
  • SQL虽然看上去很简单，但实际上是一种强有力的语言，灵活使用其语言元素，可以进行非常复杂和高级的数据库操作。

SQL应用

SQL作为一种操作数据库数据的编程语言，其作用就是操作数据库中海量的数据，尤其是在目前的互联网环境下，巨量数据的产生，在数据挖掘前期的数据清洗和加工过程中，一定程度上都离不开SQL。

数据表结构介绍

1.数据类型
数据表的数据类型从类别上分为以下几类：

  • 字符型
  • 浮点型
  • 整型
  • 日期时间型

1.1 字符型

以上常用的字符型类型包括 CHAR(M) （M最大为255）、VARCHAR(M)、TEXT （长度为65535）、ENUM('value1', 'value2', ...)、SET('value1', 'value2', ...)。

其中 ENUM('value1', 'value2', ...) 为枚举类型，表示只能有一个值的字符串，从值列’value1’，’value2’，…，NULL中或特殊’’错误值中选出。ENUM列最多可以有65,535个截然不同的值。ENUM值在内部用整数表示。 SET('value1', 'value2', ...) 为一个设置。字符串对象可以有零个或多个值，每个值必须来自列值’value1’，’value2’，…SET列最多可以有64个成员。SET值在内部用整数表示。

1.3 整型

整型类型的选择影响到内存的优化。一般来说， TINYINT 用于0-255以内的数值（无符号数，有符号数对半），不是很大的数据可以采用 SMALLINT （无符号数为65536长度，有符号数对半），其他三种类型可按需选择。

1.4 日期时间型
这类数据类型包括 YEAR （YYYY，1901-2155）、 TIME （HH:MM:SS）、 DATE （YYYY-MM-DD）、 DATETIME （YYYY-MM-DD HH:MM:SS）、 TIMESTAMP （YYYY-MM-DD HH:MM:SS）。数据范围此处不详细展开。

2.约束

  • 约束保证数据的完整性和一致性
  • 约束分为表级约束和列级约束(列参数后定义，一般多为列级约束)
  • 约束类型包括：
    • NOT NULL
    • PRIMARY KEY（主键约束）
    • UNIQUE KEY
    • DEFAULT
    • FOREIGN KEY（外键约束）

2.1 NOT NULL 非空约束
有 NOT NULL 与 NULL 之分，当没有 NOT NULL 时默认可以为 NULL

2.2 PRIMARY KEY 主键约束

  • 每张数据表只能存在一个主键
  • 主键保证记录的唯一性
  • 主键自动为NOT NULL

AUTO_INCREMENT 为自动编号，AUTO_INCREMENT 必须和 PRIMARY KEY 一起使用，反之则不需要

2.3 UNIQUE KEY 唯一约束

  • 唯一约束可以保证记录的唯一性
  • 唯一约束的字段可以为空值（NULL）
  • 每张数据表可以存在多个唯一约束

2.4 DEFAULT 默认值
默认值直接跟在 DEFAULT 之后即可，如 DEFAULT '3'

2.5 FOREIGN KEY 外键约束

  • 保证数据一致性，完整性
  • 实现一对一或多为多约束
