Python学习笔记1：Anaconda与IPython

发表于 2017-09-28 | 分类于 Programming

这一部分是Python方面的一些学习笔记，用于记录自己学习Python的过程，同时也是对自己Python这块知识的一个梳理，后期也可用于Python复习。

Python学习的第一篇，主要介绍第三方库Anaconda和第三方工具IPython的应用。

Python版本的选择

对于Python版本的选择，主要在于是Python2还是Python3的选择，Python社区正在慢慢地从Python2系列解释器过渡到Python3系列。因而，对于一般的Python开发和学习来说，可以选择Python3，而对一部分领域（如数据分析）来说，有一部分PythonPython科学计算社区还没有转向Python3，因而对于这一块，仍可采用Python2进行开发和学习，当然，采用Python3也可完成大部分的开发和学习内容。

之所以有Python2和Python3的区别，在于代码是否支持向后兼容。在Python3.0面世以前，所有的Python代码都是向后兼容的，为了让Python语言更加先进，Python社区认为做出一些向后不兼容的修改是必要的，因此会有Python2和Python3的区别。

如何辨别使用的代码是Python2还是Python3？一个最简单的判断是看print语法。

开发环境IDE的选择-Anaconda

  1. 原生Python的安装
    如果选择原生Python安装的话，可以前往Python官网选择对应的版本进行安装即可。其中，建议直接选择.exe可执行文件直接安装，x86对应32位系统，x86-64对应64位系统。中间还涉及到pip命令的安装，各种包（packages）的安装。在包安装的过程中，如出现问题可参照安装numpy+scipy+matlotlib+scikit-learn及问题解决文章来解决。

  2. 第三方Python工具的选择
    除了Python原生之外，还可以选择其他的Python发行版本来便于Python开发和学习。这里选择Anaconda来进行Python安装。

从介绍来说，Anaconda是数据分析方面常用的一种Python发行版本，其包含了conda、Python等190多个科学包及其依赖项。Anaconda作为一种Python发行版本，最便捷的两个地方在于：1.借助conda可以实现多版本并存、切换；2.借助conda可以实现各种开源包（packages）的管理（安装Anaconda时默认已安装了大部分的Python学习包）。

关于Anaconda安装部分不多说了，如有问题可参照网上的解决办法。安装完Anaconda之后，即可使用conda命令

另外，Anaconda安装完了之后，会有Anaconda Prompt、Anaconda Navigator、Anaconda Cloud、IPython Notebook、Jupyter Notebook、Spyter等软件出现。这些软件功能如下：

IPython

IPython是一个叫高效的交互式Python shell，除此之外，它还有一个可以直接进行回吐操作的GUI控制台，一个基于Web的交互式笔记本，一集一个轻量级的快速并行计算引擎。

启动IPython有两种方法，一可以直接启动安装Anaconda之后显示的Ipython Notebook，二也可以通过Anaconda Prompt命令行中执行ipython进入IPython界面。

下面讲述IPython操作的一些技巧和功能：

1. 与操作系统进行交互

IPython可以直接和系统shell进行交互。以下是与系统相关的Ipython模式命令

  • !cmd 在系统shell中执行cmd命令
  • %alias alias_name cmd 为系统shell命令定义别名
  • %bookmark 使用IPython的目录书签系统
  • %cd dierctory 更改工作目录
  • %pwd 显示当前动作目录
  • %push directory 将当前目录入栈，并转向目标目录
  • %popd 弹出栈顶目录，并转向该目录
  • %dirs 返回一个含有当前目录栈的列表
  • %dhist 打印目录访问历史记录
  • %env 以dict形式返回系统环境变量

2. 标准键盘命令

与普通cmd窗口不同，IPython可以执行标准键盘命令。常见标准键盘命令如下：

  • Ctrl+V / Ctrl+Shift+V : 粘贴，可跨行粘贴完整代码
  • Ctrl+C: 中止正在执行的代码
  • Ctrl+K: 删除从光标开始至行尾的文本
  • Ctrl+U: 清空当前航的所有文本（和Ctrl+K相反，即删除从光标开始至行首的文本，并非完全删除）
  • Ctrl+L: 清屏

3. Tab键自动补全

Tab键可以实现自动补全功能，其中包括已输入变量（对象、函数）的补全，模块的补全

4. 内省

内省是指显示变量/对象的一些通用信息。在变量的前面或者后面加上一个问号?，就可以将有关该对象的一些通用信息显示出来。??两个问号则会显示函数的source源代码部分。

从图中可以看出，?一个问号会显示docstring部分，??两个问号则会显示source部分，而没有docstring部分。其他部分两个命令相同。

注：对于不同的对象，如变量和函数两种对象，内容的结果会有所区别，因为其通用属性不同。

5. %run命令执行文件

In[1]: %run test.py

这种方式需要注意的是工作路径，默认采用的是默认路径，如涉及到其他路径，可以通过cd命令切换到其他路径，或者run的命令中写明绝对路径。

%run命令执行后，文件中所定义的全部变量（含import、函数、全局变量）就可以在当前IPython shell中直接调用了。

6. 执行剪贴板中的命令

%paste可以实现执行剪贴板中的命令。%paste回车后直接执行剪贴板代码，早期的时候有个%cpaste命令，%cpaste命令则允许多次粘贴然后执行粘贴的代码。但查阅了IPython Magic文档后发现%cpaste命令应该是被取消了。

说明，执行%cpaste后，无法手动 Ctrl+V 粘贴，也无法鼠标右键进行粘贴，似乎只能手动输入，同时其也可以与剪贴板无关系，如下图In[21]处代码，此部分代码为手动输入，执行的结果与剪贴板无关，In[22]命令为%cpaste中调用%paste命令执行结果。

%time命令用于计算单次命令运行时间，%timeit则会自动多次执行命令函数输出运行的平均时间。

8. Magic魔术命令

下述列举部分magic命令：

  • %magic 显示所有魔术命令的文档
  • %debug 从最新的异常跟踪的地步进入交互式调试器
  • %hist 打印命令的输入、输出历史记录
  • %pdb 在异常发生后自动进入调试器
  • %paste 执行剪贴板中的python代码
  • %reset 删除interactive命名空间中的全部变量/名称
  • %run filename.py 在IPython中执行一个python脚本文件
  • %prun statement 通过cProfile执行statement，并打印分析器输出结果
  • %time statement 报告statement运行时间
  • %timeit statement 自动多次执行statement以计算平均运行时间

其中，%hist用于输出历史记录（此时也可用hist），同时，_、__、___和i、ii、_iii变量保存最后三个输出和输入对象。_n和_in(n表示具体的数字)变量返回第n个输出和输入的历史命令（输出可选，无输出时不显示输出）。

9. 使用pylab进行交互式计算

对于Numpy和matplotlib中的科学计算功能，pylab命令可以使其在控制台进行交互式计算和动态绘图，对于绘图的直观影响是，不需要通过执行plt.show()看到输出的结果图。

已经进入IPython界面，可以执行%pylab实现交互式，而在最开始进入IPyhton界面之前，直接输入ipython --pylab命令，则可以进入自动带交互式计算和绘图的IPython界面。

10. IPython notebook

IPython notebook是一种基于WEB的交互式笔记本，它是一种基于JSON的文档格式.ipynb，可以轻松分享代码、输出结果及图片等内容。

执行ipython notebook即可启动IPython notebook。如IPython的pylab一样，pylab既可以在运行 ipython notebook之前输入，也可以在运行ipython notebook之后输入。运行前，ipython notebook --pylab=inline，运行后，%pylab inline。

同时，IPython notebook不仅可以作为一个python工具，还可以记录md文本等，完全可以作为一个笔记本使用。

IPython notebook输出结果可分为两种。

  • Ctrl+Enter显示当前Cell行的运行结果
  • Shift+Enter运行当前Cell行所有结果并新建一个cell输入框。
